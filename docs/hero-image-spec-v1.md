# Hero Image Spec v1 — 슬라이드형 Hero 배경 광고 아트디렉션 설계서

> **v1.1 변경(제품 중심 재설계)**: 1차 검증에서 "모델/고민 중심" 결과가 나와, 그건 Hero가 아니라 **Problem(공감) 패턴**으로 재분류. Hero는 **제품이 주인공**(면적 25~35%), 모델은 **보조**(30~45%, 얼굴 최소). 화장품 `visual_priority`를 `[text_zone, product, model]`로, 시선 흐름을 `텍스트존→제품→모델`로 수정. 텍스트존은 "빈칸"이 아니라 "카피 올릴 자리로 설계된 여백"으로 재정의.

> **이 문서의 성격**: 프롬프트 모음이 아니라 **광고 아트디렉션 설계서**다.
> Gemini · Flux · Midjourney · GPT Image — 어떤 이미지 모델을 쓰더라도 이 문서 하나로 **동일한 Hero 구조**를 재현할 수 있어야 한다.
> **Prompt First ❌ / Art Direction First ⭕**
>
> 기준 레퍼런스: **리프그린 시카토너 Hero**

---

## ★ 전제 (반드시 먼저 읽을 것)

Flik 슬라이드형 Hero는 **Overlay 구조**로 간다.

- **이미지 모델(Gemini 등)은 "텍스트 없는 배경 + 텍스트존(여백)"만 생성한다.**
- **실제 텍스트(헤드라인·서브카피·뱃지)는 프론트엔드 코드가 Pretendard 폰트로 이미지 위에 얹는다(overlay).**
- 따라서 이미지 모델은 **글자를 단 한 자도 그리면 안 된다**(한글 깨짐·재생성 과금의 원인).

그래서 Hero 이미지의 **성공 기준이 바뀐다**:

> ❌ "텍스트가 예쁘게 들어간 이미지"
> ⭕ **"텍스트를 넣을 공간이 완벽히 설계된 광고 배경"**

이 전제를 놓치면 이 문서 전체를 잘못 읽는다. 아래 모든 규칙은 **"비워진 여백 + 그 여백을 광고로 만드는 배경"**을 설계하기 위한 것이다.

### 코드와의 대응(현황 — 본 문서는 코드를 수정하지 않음)
- 현재 overlay 텍스트존 지시는 이미지 라우트의 `textZone:'top'` 분기에 일부만 존재한다(상단 ~40% negative space, 하단 ~60% 제품). 이 문서는 그 지시의 **상위 설계 문서**다.
- 즉 이 문서는 향후 `buildSlideImageRules`(슬라이드 이미지 규칙 빌더)의 **설계 원천**으로, 8장의 Image Brief 스키마는 그 함수에 그대로 이식 가능한 형태로 제공한다.

---

## 현재 실패 원인

`buildSlideImageRules`가 **사실상 비어 있다**(슬라이드 전용 아트디렉션 규칙이 부재/산재). 그 결과:

- 모델이 **광고 이미지**가 아니라 **상품사진(제품 누끼/평범한 스튜디오컷) 수준**을 생성한다.
- 텍스트존(여백)이 설계되지 않아 overlay 텍스트가 제품·얼굴 위에 얹히거나, 여백이 비어 허전하다.
- "좋다/나쁘다"를 **수치로 판정할 기준이 없다.**

이 문서는 위 세 가지를 한 번에 해결한다: **목적 정의 → 정량 레이아웃 → 카테고리 규칙 → 이식 가능한 스키마 → 정량 합격 기준.**

## 1. Hero의 목적

Hero는 상세페이지의 첫 화면이자 광고의 첫 1초다. 다음 5가지를 **동시에** 수행해야 한다.

| 목적 | 설명 | 이미지가 책임지는 부분 |
|---|---|---|
| **신뢰 확보** | "이건 제대로 된 브랜드다"라는 인상 | 광고급 조명·구도·여백(아마추어 상품컷 탈피) |
| **브랜드 인지** | 톤·색·무드로 브랜드를 각인 | 일관된 색감·무드보드 적용 |
| **제품 이해** | "무엇을 파는지" 0.5초 안에 | 제품이 식별 가능한 크기·선명도로 존재 |
| **감정 전달** | target_desire/감정 목표를 시각으로 | 모델 표정·장면·라이팅의 정서 |
| **스크롤 유도** | 다음 섹션으로 시선을 흘려보냄 | 시선 흐름 설계(아래 3장)로 하단 유도 |

> 핵심: Hero는 "제품을 보여주는 사진"이 아니라 **"제품을 사고 싶게 만드는 광고 한 장"**이다.

## 2. 레이아웃 구조 분석 (제품 중심 Hero, 정량)

> ★ **Hero vs Problem 재분류 (v1.1 핵심 수정)**
> **모델/고민 중심 구도(손가락 볼 터치·피부 클로즈업이 화면을 지배)는 Hero가 아니라 Problem(공감) 패턴이다.**
> Hero는 "제품·브랜드 첫인상" → **제품이 주인공**이어야 한다. 모델은 있어도 **제품을 받쳐주는 보조**(얼굴 노출 최소).
> (모델 지배 레이아웃이 필요하면 그건 Problem 섹션 스펙으로 분리한다.)

Hero 프레임을 4개 존으로 나눈다. **세로 비율(%)은 프레임 높이 기준**, 제품/모델 %는 **피사체가 차지하는 시각적 면적 기준**(존과 측정 기준이 다르므로 합이 100을 넘을 수 있음 — 존은 위치, 면적은 크기다).

```
┌───────────────────────────────┐
│  ● 추천상품 (뱃지)              │
│  큰 헤드라인                    │  ← [텍스트존] 상단 35~40% (카피 올릴 자리로 '설계된' 여백)
│  서브카피 한 줄                 │     ※ Overlay 전용. 이미지엔 글자 없음
│┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄│
│        ┌──────────┐           │
│   (보조 │  제 품    │           │  ← [제품=주인공] 중앙~하단, 면적 25~35%, 크고 선명
│   모델) │  (주인공) │           │     [모델=보조] 제품 뒤/옆 받침, 얼굴 노출 최소
│        └──────────┘           │
│  · 무향 · 저자극 · 테스트완료   │  ← [기능영역] 하단 10~15% (아이콘/심볼, 글자 아님)
└───────────────────────────────┘
```

| 존 | 세로 위치 | 비중 | 측정 기준 | 비고 |
|---|---|---|---|---|
| **텍스트존** | 상단 | **35~40%** | 프레임 높이 | **Overlay 전용 = 카피 올릴 자리로 설계된 여백**(빈칸이 아니라 의도된 공간). 실제 텍스트 없음 |
| **제품(주인공)** | 중앙~하단 | 면적 **25~35%** | 피사체 면적 | **Hero의 주인공.** 크고 선명·식별. 5장·6장 규칙 |
| **모델(보조)** | 제품 뒤/옆 | 면적 **30~45%**, 얼굴 노출 최소 | 피사체 면적 | 제품을 받쳐주는 보조. 화면 지배 금지(지배 시 Problem임). 없어도 됨(4장) |
| **기능영역** | 하단 | **10~15%** | 프레임 높이 | 심볼/아이콘/질감 암시(글자 금지) |

> ※ **텍스트존은 "무엇도 그리지 않는 곳"**이되, "허전한 빈칸"이 아니라 **"카피가 얹힐 것을 전제로 설계된 여백"**이다(상단 35~40%, 좌측, 저대비·균일). 깨끗한 soft gradient/플레인 배경으로 정의한다.

## 3. 시선 흐름(Z/F) 설계

광고는 "보는 순서"를 설계한다. Hero의 의도된 시선 경로:

```
① 텍스트존(상단 여백)  ─ overlay 헤드라인이 얹힐 자리, 가장 먼저 인지
        ↓
② 제품(중앙~하단)      ─ "이걸 파는구나" 제품이 주인공, 첫인상 확정
        ↓
③ 모델(보조)          ─ 제품을 받쳐주는 감정·맥락(얼굴 노출 최소)
        ↓
④ 기능요약(하단)      ─ 근거/안심, 그리고 다음 섹션으로 스크롤
```

**왜 광고에서 이 순서가 중요한가**
- 상단→하단 자연 시선과 일치(F패턴): 메시지(텍스트존)→제품→맥락→근거가 순서대로 들어온다.
- **Hero는 "제품·브랜드 첫인상"** → 제품(②)을 먼저 확정시키고, 모델(③)은 그 제품에 감정·맥락을 더하는 보조다. (모델이 먼저/지배하면 그건 Hero가 아니라 Problem 공감컷이다.)
- 마지막 ④가 **하단에 있어 스크롤을 유도**한다(시선이 화면 밖 아래로 빠져나감 = 다음 섹션 진입).
- 배경/구도는 이 흐름을 **방해하지 않도록**: 강한 대비나 시선 끄는 오브젝트를 ②③④ 경로 밖(특히 텍스트존)에 두지 않는다.

## 4. 카테고리별 Hero 구조

> ⚠️ **모델 정책 — 절대 규칙**
> - **일관된 모델 얼굴 시스템 금지.** 같은 가상 인물을 페이지/섹션마다 고정 출연시키지 않는다.
> - **가상모델 고정 운영 금지.** "우리 브랜드 전속 AI 모델" 식의 식별 가능한 동일 얼굴 반복 금지.
> - 모델이 필요할 때도 **식별되는 정면 얼굴 화보 대신** 신체 일부·피부 클로즈업·뒷모습·실루엣 등 **"감정/상황 전달용"**으로 쓴다.
> - 아래 "모델 필요 여부"는 이 제약 안에서의 권장이다.

| 카테고리 | 모델 필요 | 제품 강조 비율(면적) | 촬영 방식 | 감정 목표 | 추천 구도 |
|---|---|---|---|---|---|
| **1. 화장품** | △ **보조만**(질감/제형/피부 일부, 얼굴 노출 최소·화보 금지) | **25~35%(주인공)** | **제품 클로즈업이 중심** + 질감/제형, 소프트 라이팅 | 안심·깨끗함·진정 | 상단 여백 / 중앙~하단 제품(크게) / 모델은 제품 뒤 받침 |
| **2. 건강기능식품** | ✕ (모델 없음 권장) | 25~35% | 제품 단독 + 원료 심볼(허브/캡슐), 클리니컬 | 신뢰·전문성·근거 | 상단 여백 / 중앙·하단 제품+원료 / 데이터 톤 |
| **3. 식품** | ✕ (음식이 주인공) | 제품 20~30% + 음식 30~40% | 푸드 스타일링, 식감·신선도 클로즈업 | 식욕·신선·따뜻함 | 상단 여백 / 중앙 음식 단면·김 / 제품 패키지 하단 |
| **4. 반려동물** | △ (반려동물=모델, 사람 얼굴 X) | 18~25% | 반려동물 + 제품, 따뜻한 자연광 | 애정·안심·건강 | 상단 여백 / 중앙 반려동물 / 좌하단 제품 |
| **5. 생활용품** | ✕ (공간/사용맥락이 모델) | 20~30% | 제품 + 사용 공간(맥락컷), 미니멀 | 깔끔·편리·만족 | 상단 여백 / 중앙 제품+공간 / 하단 디테일 |

**카테고리별 "모델 사용 vs 제품 단독" 기준**
- **★ Hero 공통 원칙**: Hero는 어떤 카테고리든 **제품이 주인공**(면적 25~35%). 모델/생물은 **보조**로만 쓰며 화면을 지배하면 안 된다(지배하면 Problem 공감컷).
- **모델/생물 보조 사용**: 감정 이입이 도움 되는 카테고리(화장품 피부, 반려동물, 식품 식욕). 단 사람은 **얼굴 화보 금지, 부분/상황 컷**으로, 제품 뒤/옆 받침 역할.
- **제품 단독**: 신뢰·스펙·근거가 구매를 좌우하는 카테고리(건기식, 생활용품). 모델 없이 제품+원료/맥락으로.

## 5. 텍스트존 규칙 (Overlay 전제)

텍스트존은 **이미지에서 비워두는 여백**이며, 프론트가 Pretendard 텍스트를 얹는 공간이다.
**단 "허전한 빈칸"이 아니라 "카피가 얹힐 것을 전제로 설계된 여백"**이다 — 상단 35~40%에, 좌측 정렬 텍스트가 들어갈 폭/높이를 확보하고, 그 자리는 저대비·균일하게 의도적으로 비운다. (비어 보이면 실패, "여기 카피가 올라가겠구나"가 보이면 성공.)

| 항목 | 규칙 |
|---|---|
| **위치** | 상단 **35~40%** 고정(매번 같은 위치 — AI가 임의로 다른 곳 비우지 않게) |
| **정렬** | **좌측 정렬 우선**(리프그린 기준). 중앙은 카테고리 예외만 |
| **최소 크기** | 프레임 폭의 좌측 ~70%, 높이 상단 35% 이상 확보 |
| **명도** | 텍스트존 배경은 **밝고 균일**(어두운 텍스트가 얹힘). 또는 일관된 라이트 톤 |
| **여백** | 가장자리 안전 여백 확보(텍스트가 프레임 끝에 붙지 않게 상/좌 패딩 영역) |
| **가독성** | 텍스트존 배경은 **저대비·저디테일**. overlay 글자가 항상 읽혀야 함 |

**텍스트존 금지 사항**
- ❌ 텍스트 뒤 복잡한 오브젝트
- ❌ 텍스트 뒤 제품
- ❌ 텍스트 뒤 얼굴
- ❌ 텍스트 뒤 강한 명암/하이라이트
- ❌ 텍스트존에 이미지 모델이 글자를 그려 넣는 것(절대 금지)

## 6. 제품 배치 규칙

제품은 광고의 주인공이다. "있는 듯 없는 듯"은 실패다.

**필수 규칙**
- **최소 면적 비율**: 프레임의 **18% 이상**(건기식·생활용품·식품 패키지는 25%+ 권장).
- **최소 크기**: 라벨·형태·색이 **식별 가능**한 크기. 모바일 썸네일에서도 "무엇인지" 보여야 함.
- **배치 우선순위**: 좌하단 → 중앙하단 → 우하단(텍스트존과 겹치지 않는 하단 영역).
- **선명도**: 제품은 초점이 맞고 또렷하게(배경보다 우선).

**금지 사례**
- ❌ 제품이 화면 5% 이하
- ❌ 제품이 모델보다 지나치게 작음
- ❌ 제품이 배경에 묻힘(색·명도 대비 부족)
- ❌ 제품 식별 불가(흐림/잘림/뒤돌아 있음)

## 7. 무드보드 정의

무드는 카테고리·브랜드 톤에 따라 1개 선택. 각 무드는 색·조명·배경·소품·인상을 규정한다.

| 무드 | 색감 | 조명 | 배경 | 소품 | 브랜드 인상 |
|---|---|---|---|---|---|
| **1. 내추럴** | 그린·베이지·웜뉴트럴 | 부드러운 자연광, 그림자 약함 | 린넨/우드/식물 | 잎·물방울·돌 | 순하고 건강한 |
| **2. 클리니컬** | 화이트·아이스블루·실버 | 균일·평평한 라이트, 그림자 최소 | 클린 화이트/그라데이션 | 유리·물·미니멀 도구 | 검증된·전문적 |
| **3. 프리미엄** | 딥톤·차콜·골드 포인트 | 방향성 있는 키라이트, 깊은 음영 | 다크/텍스처 표면 | 패브릭·스톤·반사 | 고급·묵직한 |
| **4. 미니멀** | 모노톤·단색·소프트 파스텔 | 균일·플랫, 그림자 거의 없음 | 단색 평면 | 거의 없음(여백 강조) | 깔끔·현대적 |

> 무드는 **전 섹션 공통**으로 유지해야 브랜드 인지가 쌓인다(Hero에서 정한 무드를 페이지가 따른다).

## 8. Image Brief 스키마 (`buildSlideImageRules` 이식용)

**프롬프트 문장 금지.** 값은 enum/구조화 데이터로. 이 스키마를 `buildSlideImageRules`가 받아 모델별 프롬프트로 변환한다.

```jsonc
{
  "purpose": "hero",                         // 고정: 이 브리프는 Hero용
  "emotion": "안심 | 신뢰 | 식욕 | 애정 | 깔끔",   // target_desire 기반 1개

  "visual_priority": ["text_zone", "product", "model"],  // ★ 필수 — Hero=제품 우선(모델은 보조)

  "layout": {
    "text_zone":      { "position": "top",  "ratio": 0.38, "align": "left" },   // 35~40%
    "product_position": { "position": "center-lower", "area_min": 0.25 },        // 주인공 25~35%
    "model_position": { "position": "behind-product", "band": [0.45, 0.90], "role": "support" }   // 보조, 없으면 null
  },

  "text_zone": {
    "location": "top",            // top | bottom (Hero v1 = top)
    "min_ratio": 0.38,            // 프레임 높이 비율(35~40%)
    "align": "left",
    "intent": "reserved-for-copy",// 빈칸이 아니라 '카피 올릴 자리로 설계된 여백'
    "background": "light-uniform",// light-uniform | soft-gradient
    "forbid": ["product", "face", "high-contrast", "busy-objects", "text"]
  },

  "product_placement": {
    "required": true,
    "area_min": 0.25,             // 면적 최소(주인공) 25~35%
    "priority": "center-lower",
    "identifiable": true,
    "in_focus": true,
    "is_hero": true               // Hero의 주인공
  },

  "model_use": {
    "enabled": true,              // 카테고리 규칙(4장). Hero에선 보조 역할
    "role": "support",            // support(제품 받침) — Hero에서 화면 지배 금지
    "type": "partial",           // partial(피부/손/뒷모습) | subject(반려동물/음식) | none
    "face_exposure": "minimal",  // 얼굴 노출 최소(눈·정면 화보 금지)
    "forbid_consistent_face": true,  // ★ 동일 얼굴 고정 금지
    "forbid_virtual_model_lock": true
  },

  "background": "natural | clinical | premium | minimal",  // 무드와 정합
  "lighting":  "soft-daylight | flat-even | directional-key | flat-soft",
  "mood":      "natural | clinical | premium | minimal",

  "negative": [
    "any text, letters, numbers, typography",   // 글자 금지(overlay 전제)
    "logos/badges/certification marks",
    "product smaller than area_min",
    "product blending into background",
    "busy/high-contrast objects in text_zone",
    "recognizable repeated model face",
    "fake data/graphs/percentages"
  ]
}
```

### ★ `visual_priority` 필수 — 카테고리별 우선순위

이미지 모델이 **무엇을 먼저/크게 배치할지** 결정하는 핵심 배열. 앞에 올수록 우선.

```jsonc
// 화장품  (v1.1: 제품을 모델 위로 — Hero=제품 첫인상)
["text_zone", "product", "model"]

// 건강기능식품
["text_zone", "product"]          // 모델 없음 → 제품 우선

// 식품
["product", "food"]               // 음식·제품이 주인공(텍스트존은 여전히 상단 확보하되 우선순위는 제품)

// 반려동물  (제품 우선, 반려동물은 보조)
["text_zone", "product", "model"] // model = 반려동물(보조)

// 생활용품
["text_zone", "product"]          // 제품 + 공간 맥락
```

> 식품처럼 `text_zone`이 배열 앞에 없어도 **텍스트존(상단 여백)은 항상 확보**한다. `visual_priority`는 "주피사체 강조 순서"이고, 텍스트존 확보는 `layout.text_zone`/`text_zone`이 별도로 강제한다.

## 9. 성공 사례 분석 — 리프그린 Hero 해부

"왜 이건 광고처럼 보이는가"를 **수치로** 분해한다. 목표는 "좋아 보인다/안 좋아 보인다"가 아니라 **수치 기반 평가**다.

| 항목 | Hero 목표 수치(v1.1) | 광고가 되는 이유 |
|---|---|---|
| **제품 크기** | 면적 **25~35%(주인공)** | Hero=제품 첫인상. 크고 선명하게 식별 |
| **모델 비율** | 면적 **30~45%(보조)**, 얼굴 최소 | 제품을 받쳐주는 감정·맥락(지배 시 Problem) |
| **텍스트존** | 상단 **35~40%** (카피 올릴 자리로 설계) | overlay 메시지가 항상 읽힘 + 광고 레이아웃 |
| **여백** | 텍스트존+가장자리 안전여백 확보 | "숨 쉬는" 구성 = 프리미엄 인상 |
| **시선 흐름** | 텍스트존→제품→모델→기능 (상→하) | 제품 먼저(첫인상), 모델은 보조, 스크롤 유도 |
| **배경** | 저대비·단순(무드 일관) | 제품·텍스트를 방해하지 않음 |
| **감정** | 진정/안심(소프트 라이팅·그린톤) | target_desire와 정합 |

**정량 합격 밴드(요약)**
- 제품 면적: **25~35%(주인공)** (건기식/생활도 25~35%)
- 모델(보조): **30~45%**, 얼굴 노출 최소 (지배 금지)
- 텍스트존: **35~40%** (상단, 카피 올릴 자리로 설계된 여백)
- 기능영역: **10~15%** (하단)

## 10. 실패 체크리스트 (객관 판정용)

현재 실패 Hero와 비교 가능한 체크리스트. 하나라도 ✓면 재작업.

- □ 제품이 너무 작다 (면적 25% 미만 — Hero 주인공 기준)
- □ 모델/사람이 화면을 지배한다 (= Hero가 아니라 Problem 공감컷)
- □ 광고가 아니라 상품사진 같다
- □ 텍스트존(상단 여백)이 없다 / 비어있지 않다
- □ 브랜드 감정이 없다 (무드/정서 부재)
- □ 시선 흐름이 없다 (상→하 경로 무너짐)
- □ 모델/구도가 광고스럽지 않다
- □ 상단 35~40%가 비워져 있지 않다 (제품/얼굴/오브젝트 침범)
- □ 제품 식별이 어렵다 (흐림/잘림/뒤돌아 있음)
- □ 배경이 제품보다 눈에 띈다 (배경 과잉)
- □ 이미지에 글자/로고/숫자가 그려져 있다 (overlay 전제 위반)

### ★ 가장 중요한 단일 판정

> **"텍스트(overlay)를 제거해도 광고처럼 보이는가?"**
>
> **YES → 성공. NO → 실패.**

텍스트를 떼면 평범한 상품사진이 되는 Hero는 실패다. 배경 자체가 이미 광고여야 한다.

## 완료 기준

이 문서 하나로:

1. **Gemini · Flux · Midjourney · GPT Image** 모두 동일한 Hero 구조를 생성할 수 있다(모델 무관, Art Direction First).
2. `buildSlideImageRules` 설계의 **상위 문서** 역할을 한다(8장 스키마가 변환 입력).
3. 8장 **Image Brief 스키마를 코드에 바로 이식** 가능하다(프롬프트 문장 없음, 구조화 데이터).
4. 10장 **체크리스트 + 9장 정량 밴드**만으로 다음 Hero를 **성공/실패 판정** 가능하다.

> 본 작업: **문서만 작성.** 코드 수정 없음 · 프롬프트 수정 없음.
